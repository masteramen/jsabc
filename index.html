Spring Overview

1. What is Spring?

Spring is an open source Java EE development framework.

2.What are the advantages of Spring?

  • Lightweight: Spring is definitely lightweight in terms of size and transparency, and the basic version of the Spring framework is only about 2MB.

  • Inversion of Control (IOC): Spring uses the control reversal technique to achieve loose coupling.

  • Aspect-oriented programming (AOP): Spring supports aspect-oriented programming while separating the application's business logic from the system's services.

  • Containers: Spring includes and manages the configuration and lifecycle of application objects.

  • MVC framework: Spring's web framework is a well-designed web MVC framework that replaces some web frameworks.

  • Transaction management: Spring provides a unified transaction management interface for down to local business to global services (JAT).

  • Exception Handling: Spring provides a handy API to turn specific technology exceptions (thrown by JDBC, Hibernate, or JDO) into a consistent, Unchecked exception.

Dependency Injection

18. What is dependency injection in Spring?

Dependency injection as a layer of control over inversion (IOC) can be interpreted in various ways.

19. What are the different types of IOCs (Dependent Injection)?

  • Constructor Dependency Injection: Constructor dependency injection is done when the container triggers the constructor. The constructor has a series of parameters, each of which represents the injected object.

  • Setter method dependency injection: First, the container triggers a parameterless constructor or an objectless static factory method to instantiate the object, and then the container calls the setter method of the bean to complete the setter method dependency injection.

20. What kind of dependency injection do you recommend?

You can use both methods of dependency injection at the same time. The best choice is to use constructor parameters to implement mandatory dependency injection, and setter methods to implement optional dependencies.

Spring Beans

21. What is Spring Beans?

Spring Beans is the Java object that forms the core of the Spring application.

Beans created in Spring are singleton beans.

22. What is defined in Spring Bean?

The Spring Bean defines all the configuration metadata that tells the container how to create it, what its life cycle is, and its dependencies.

23. How to provide configuration metadata to the Spring container?

There are three ways to provide metadata to the Spring container:

  • XML configuration file

  • Based on annotation configuration

  • Java-based configuration

24. How do you define the scope of the bean?

When creating a bean in Spring, we can declare its scope.

25. Talking about the bean scope supported by Spring

The Spring Framework supports the following five different scopes:

  • Singleton: There is only one bean instance in the Spring IOC container, and the bean exists as a singleton.

  • Prototype: A bean can define multiple instances.

  • Request: A new bean will be created for each HTTP request.

  • Session: A HTTP session defines a bean.

  • globalSession: Defines a bean with the same global HTTP Session.

The bean's default scope attribute is 'singleton'.

26. Is the singleton beans in the Spring framework thread-safe?

No, singleton beans in the Spring framework are not thread-safe.

27. Explain the life cycle of beans in Spring framework

  • The Spring container reads the bean definition in the XML file and instantiates the bean.

  • Spring sets the property value according to the bean's definition.

  • If the bean implements the BeanNameAware interface, Spring passes the bean's id to the setBeanName() method.

  • If the bean implements the BeanFactoryAware interface, Spring passes the beanfactory to the setBeanFactory() method.

  • If any bean BeanPostProcessors is associated with this bean, Spring calls the postProcessBeforeInitialization() method.

  • If the bean implements the InitializingBean interface, call the afterPropertiesSet method in the bean.

  • If any bean BeanPostProcessors is associated with this bean, call the postProcessAfterInitialization() method.

  • If the bean implements DisposableBean, call the destroy() method.

28. Which are the most important bean lifecycle methods?

There are two important bean lifecycle methods.

The bean tag has two important attributes (init-method and destroy-method). You can define your own initialization methods and destructor methods through these two attributes.

29. What is Spring's internal bean?

When a bean is used as a property of another bean, the bean can be declared as an inner bean.

30. How to inject Java collection class in Spring?

Spring provides the following types of collection configuration elements:

  • The list element is used to inject a series of values, allowing the same value.

  • The set element is used to inject the values ​​of some columns, which are not allowed to have the same value.

  • Map is used to inject a set of key-value pairs. The key and value can be any type.

  • Props can also be used to inject a set of key-value pairs, where the keys and values ​​are all string types.

31. What is a bean wiring?

Wiring, or bean Wiring, refers to the situation where beans are combined in a Spring container.

32. What is the automatic assembly of beans?

Spring containers can automatically configure the relationships between the cooperating beans.

33. Explain various modes of automatic assembly?

Automatic assembly provides five different modes for the Spring container to use for automatic dependency injection between beans:

  • No: The default method is to not auto-assemble and assemble the bean by manually setting the ref attribute.

  • byName: automatic assembly by parameter name, Spring container finds the properties of beans, which are set to byName in the XML configuration file.

  • byType: Auto-assembles automatically through the data type of the parameter. The Spring container looks up the properties of the beans. These beans are set to byType in the XML configuration file.

  • Constructor: This is similar to byType, but is a parameter that is applied to the constructor.

  • Autodetect: If there is a default constructor, it will be assembled automatically by means of construct, otherwise it will be automatically assembled using byType.

34. What are the limitations of automatic assembly?

Automatic assembly has the following limitations:

  • Rewrite: You still need to specify dependencies using the and <property> settings, which means that you always have to override auto-assembly.

  • Native data types: You cannot automatically assemble simple properties such as native types, strings, and classes.

  • Fuzzy characteristics: Automatic assembly always has no custom assembly precision, so if possible, use a custom assembly.

Spring annotations

36. What is Spring's Java-based configuration?

The Java-based configuration allows you to use Java's annotations for most of Spring's configuration rather than the traditional XML file configuration.

Take the annotation @Configuration as an example. It is used to mark the class as a bean definition that can be used by the Spring IOC container.

37. What is an annotation-based container configuration?

Another alternative to XML configuration is annotation-based configuration, in which byte metadata assembly components are used instead of angle brackets.

38. How to open an annotation assembly?

Annotation assembly is not enabled by default in Spring containers.

39.The @Required annotation

@Required indicates that the properties of the bean must be set at configuration time and can be explicitly specified in the bean's definition or through automatic assembly settings.

40.@Autowired annotation

The @Autowired annotation provides more granular control, including where automatic assembly is done and how it is done.

41. The @Qualifier annotation

When there are multiple beans of the same type and only one of them needs to be assembled automatically, use the @Qualifier annotation in conjunction with the @Autowire annotation to eliminate this confusion and indicate the beans that need to be assembled.

48. Spring supported transaction management types

Spring supports transaction management in two ways:

  • Programmatic transaction management: This means that you can manage transactions programmatically, which brings great flexibility but is difficult to maintain.

  • Declarative transaction management: This approach means that you can separate transaction management from business code.

49. What are the advantages of transaction management in the Spring framework?

  • It provides a unified programming model for different transaction APIs such as JTA, JDBC, Hibernate, JPA, and JDO.

  • It provides a simple API for programmatic transaction management rather than a series of complex transaction APIs (such as JTA).

  • It supports declarative transaction management.

  • It can be well integrated with Spring's multiple data access technologies.

50. What type of transaction management do you recommend?

Many users of the Spring Framework choose declarative transaction management because it is less relevant to applications and therefore more consistent with the concept of lightweight containers.

Spring Aspect-Oriented Programming (AOP)

51. Explain AOP

Aspect-oriented programming, or AOP, allows programmers to modularize horizontal business logic or define core functionality such as log management and transaction management.

52. Aspect

The core of AOP is the aspect, which encapsulates the generic behavior of multiple classes as reusable modules.

53. What is the difference between concern and cross-cutting concern in Spring AOP?

Concern (core logic): Indicates the behavior of a module in an application.

Cross-cutting concern: Refers to functionality that is used throughout the application and affects the entire application.

54. Join point

The connection point represents the location in the application where the AOP aspect is inserted.

55. Advice

Notifications indicate actions that need to be performed before and after the method is executed.

Spring section can perform five types of notifications:

  • Before (pre-notice): A notification that is performed before a method.

  • After (final notification): A notification (whether it is a normal return or an abnormal exit) to be executed when a connection point exits.

  • After-returning: A notification that is executed after a connection point has completed normally.

  • After-throwing (exception notification): A notification that is executed when the method throws an exception exit.

  • Around (Circle Notification): A notification that fires before and after the method call.

56. Pointcut

A pointcut is a point or group of connection points where notifications will be performed.
